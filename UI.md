# UI Improvement Plan

## High Impact
1. **Dashboard — excessive vertical scrolling.** The 4 `StudentList` sections at the bottom (Recent, Trials, Frozen, Inactive) create a very long page. Consider collapsible sections, a tabbed layout, or only showing Recent + Trials by default with a "View all" link.
2. ~~**Loading states are inconsistent.** Dashboard has a polished skeleton loader, but Memberships, Attendance, and Schedule just show `<div>Loading...</div>`.~~ ✅ Added skeleton loaders to Memberships, Attendance, and Schedule pages.
3. ~~**Students table — no empty state.** When there are zero students, the table headers render alone with no rows.~~ ✅ Added centered empty state with icon, message, and Add Student/Import CTAs.
4. ~~**Attendance page — "No Active Class" alert uses `variant="destructive"`.** This is a normal operational state, not an error.~~ ✅ Changed to neutral `Alert` (no variant).
5. ~~**Schedule — day columns have no bottom padding or visual closure.** The time grid just stops.~~ ✅ Added `bg-muted/20 border-t` footer bar below the grid.

## Medium Impact
6. ~~**Membership plan cards — too many action buttons stacked.** Each card has View Students, Copy Link, Edit, and Delete — 4 buttons in a column. This is visually heavy. Move Edit/Delete into a `...` dropdown menu (like the Students table) and keep only the primary actions visible.~~ ✅ Edit/Delete moved into a `⋮` dropdown. View Students and Copy Link remain as primary actions.
7. ~~**StatCard trend text is hardcoded in Memberships page.** `trend="+12.5%"` and `trend="+8"` are static strings, not real data. This is misleading. Either compute real trends or remove the trend line entirely for that page.~~ ✅ Removed fake trend props from Membership page StatCards.
8. ~~**Students table pagination — plain text "Page X of Y".** The pagination feels dated compared to the rest of the UI. A proper pagination component with numbered page buttons would look more polished.~~ ✅ Replaced with numbered page buttons including ellipsis for large page counts.
9. ~~**Join page (public) — no step indicator.** The multi-step flow (Contact → Waiver → Payment) has no visual progress indicator. Adding a simple step indicator (1-2-3 dots or a horizontal stepper) would orient the student.~~ ✅ Added 3-step indicator (Contact → Waiver → Payment) with numbered circles, check marks, and connecting lines.
10. ~~**Past Due — table rows use `bg-red-50/40` which won't work well in dark mode.** Use a semantic color like `bg-destructive/5` or `data-[state=failed]:bg-destructive/5`.~~ ✅ Changed to `bg-destructive/5`.

## Polish
11. ~~**Header bar — only has a SidebarTrigger and Add Student button.** Consider adding breadcrumbs~~ ✅ Added breadcrumbs to header bar via `Layout.tsx`.
12. ~~**Sidebar nav items use `h-12` rows.** Too tall for a sidebar.~~ ✅ Reduced to `h-9` with `h-4` icons.
13. ~~**Profile page — role badge just says "Admin" or "Staff".** Should reflect actual role.~~ ✅ Now shows Owner/Admin/Coach/Front Desk.
14. ~~**Login page — no logo or branding.** Add logo mark at top.~~ ✅ Added centered logo above title.
15. ~~**Settings tabs overflow on mobile.** Consider dropdown/select for mobile.~~ ✅ Mobile shows `<Select>` dropdown, desktop shows horizontal tabs, both synced via shared state.

## Round 2
16. ~~**Students search only filters the current page.** The query fetches 10 students per page from Supabase, then `filteredStudents` filters those 10 client-side. Searching for "John" misses any John on a different page. Move search to a server-side `ilike` filter on the Supabase query, or switch to client-side pagination for the full dataset.~~ ✅ Moved to server-side `ilike` filter with 300ms debounce; page resets to 1 on search change.
17. ~~**Schedule class deletion has no confirmation dialog.** Clicking "Delete" in the schedule dropdown immediately calls the mutation with no confirmation. A single misclick permanently removes a class and its attendance history. Add an `AlertDialog` confirmation matching the pattern already used in Students and Memberships pages.~~ ✅ Added `AlertDialog` with class name and day confirmation.
18. ~~**StudentDetail still uses `<div>Loading...</div>`.** Every other page has skeleton loaders matching the final layout. StudentDetail shows unstyled text, causing a jarring content flash. Add a skeleton loader matching the StudentDetail layout (avatar circle, name line, grid of cards).~~ ✅ Added skeleton loader matching StudentDetail layout.
19. ~~**Membership plan card shows `totalMembers` as "active members".** `totalMembers` counts all students regardless of `membership_status`, but the label reads "active members." Either display `activeMembers` instead, or change the label to "enrolled members" and show `activeMembers` separately.~~ ✅ Now shows "X active · Y enrolled".
20. ~~**Dashboard active student trend is mathematically incorrect.** `activeStudentsList` only contains students whose `membership_status === 'active'` right now. Filtering by `join_date` doesn't tell you how many were active last month — students who were active then but froze/cancelled this month are excluded, making the trend artificially positive.~~ ✅ Removed misleading active student trend.
21. ~~**Attendance "Checked In" panel flashes empty while attendance data is still loading.** The page skeleton ends when student data resolves, but the attendance query may still be fetching. The panel renders "No students checked in yet" momentarily before real data arrives. Gate the checked-in list behind `isLoadingAttendance` with a small inline skeleton.~~ ✅ Added inline skeleton while attendance data loads.
22. ~~**AddStudent form has no Cancel/Back button.** The form has only a submit button. Users must use the browser back button, which is inconsistent with edit dialogs that always provide a Cancel option. Add a `variant="outline"` Cancel button.~~ ✅ Added Cancel button next to submit.
23. ~~**Sort change doesn't reset Students table to page 1.** Changing the sort column toggles `sortConfig` but leaves `currentPage` at its current value. The user may see unexpected results or a blank page. Call `setCurrentPage(1)` inside `handleSort` whenever the sort key changes.~~ ✅ Page resets to 1 on sort key change.
24. ~~**Past Due table is unusable on mobile.** 7 columns with no responsive treatment — unlike the Students table which hides columns on smaller screens. Hide lower-priority columns (Failure Reason, Next Retry) on mobile with `hidden md:table-cell`.~~ ✅ Hidden Failure Reason (md+), Retries (lg+), Next Retry (lg+).
25. ~~**Students contact column renders empty space for null email/phone.** When email or phone is null, the icon and an empty `<span>` still render, producing a blank line. Conditionally render each row only when the value exists.~~ ✅ Contact rows only render when the value exists.
26. ~~**Join page date-of-birth input accepts future dates.** The `<Input type="date">` has no `max` constraint. Add `max={new Date().toISOString().split('T')[0]}` to prevent invalid birthdates.~~ ✅ Added `max` constraint to Join and AddStudent date-of-birth inputs.
